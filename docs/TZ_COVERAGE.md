# Соответствие ТЗ: «AI-платформа глубокой переработки знаний…» (NotebookLM-style)

Оценка **по фактам репозитория** (`develop`). Шкала: **выполнено** / **частично** / **нет** + краткий комментарий.

---

## Основные задачи (обязательный контур)

| Пункт ТЗ | Статус | Реализация / замечания |
|----------|--------|-------------------------|
| **RAG:** загрузка PDF, DOCX, TXT + вопросы по содержанию | **Выполнено+** | Загрузка через `document-service`, индексация `rag-service` (`ingest`), чанки в PostgreSQL, поиск TF-IDF или эмбеддер (`EMBEDDER_BASE_URL`). **Дополнительно к ТЗ:** поддержка **PPTX** (извлечение текста для RAG). Чанкинг: режим **`semantic`** по умолчанию (смысловые границы) + `langchain` / `llama_index` / `legacy`. |
| **Чат с источниками** | **Выполнено** | `rag/query` → контекст в системный промпт → `ai/chat`. Источники в UI (чанк, файл, «релевантность» %). Для **нескольких документов** — сбалансированный поиск по каждому файлу (`ragQueryBalanced`). Страница `/chat` по всем готовым файлам + вкладка «Чат» в `/workspace/:id`. |
| **Аудиопересказ:** два ведущих, тон (научный/популярный), темп | **Выполнено*** | Сценарий диалога генерируется LLM; в UI выбор тона и темпа; **озвучка** — **Web Speech API** в браузере (демо без отдельного TTS-сервера). *Рекомендация ТЗ (Piper/Coqui) — **не реализована** как бэкенд-TTS. |
| **Формализованные отчёты** (официальный стиль, шаблон) | **Выполнено** | Вкладка «Отчёт»: структура справки (цель, содержание, положения, выводы, рекомендации) через `generateOfficialReport`. |
| **Mindmap:** интерактивный граф | **Выполнено** | Вкладка Mindmap: иерархия из LLM → `MindmapView` (интерактивный граф). |

**Итог по обязательным блокам:** ~**95–100%** по функционалу; основной **пробел относительно «рекомендаций»** — не Piper/Coqui, а браузерный TTS.

## Опциональный функционал

| Пункт ТЗ | Статус | Реализация / замечания |
|----------|--------|-------------------------|
| **Карточки (flashcards)** | **Выполнено** | Вкладка «Карточки», flip UI. |
| **Тесты с вариантами** | **Выполнено** | Вкладка «Тесты», генерация по RAG-контексту. |
| **Таблица (CSV/Excel)** | **Частично** | CSV из неструктурированного текста (`extract-table`), скачивание; **нативного XLSX** в репозитории нет (Excel открывает CSV). |
| **Пересказ видео** (картинка + звук → видео) | **Частично** | Есть **сценарий видео** (сцены, `image_hint`, тайминги), стоковые картинки; **сборка одного MP4 на сервере** в продукте не заявлена как готовая цепочка. |
| **Инфографика** | **Выполнено*** | Вкладка «Инфографика»: график по числам из RAG, JSON для экспорта. *Стабильность зависит от LLM + `json_mode` / парсинга. |
| **Презентация** | **Выполнено** | Генерация **PPTX** (pptxgenjs) по темам из источников; опционально стоковые изображения через gateway. |

## Критерии оценки (как закрывает продукт)

| Критерий | Комментарий |
|----------|-------------|
| **Качество русского** | Промпты и UI на русском; фактическое качество — от выбранной LLM и качества RAG (чанки, эмбеддер). |
| **Интеграция модулей** | Один **workspace** на документ/группу: общий `ragIds`, саммари, просто/кратко, отчёт, таблица, чат, тесты, карточки, презентация, видео-план, инфографика, mindmap, ZIP-пакет. |
| **UX/UI** | React, вкладки, загрузка, авторизация; целевой пользователь «не разработчик» — формулировка ТЗ соблюдена по задумке. |
| **Архитектурная гибкость / открытые модели** | `LLM_MODE=openai_http` + свой `MISTRAL_BASE_URL` (vLLM и т.д.); эмбеддер и STT — OpenAI-совместимые API; данные в своём Postgres + volume. **Потенциал закрытого контура** — да, через конфигурацию без обязательного интернета. |

## Требования к решению

| Требование | Статус |
|------------|--------|
| Целостная архитектура | **Да** — микросервисы, gateway, см. `README.md` |
| Рабочий алгоритм LLM через API | **Да** — `ai-service`, Mistral SDK / OpenAI-совместимый HTTP |
| Frontend + backend связаны | **Да** — Bearer, `/api/v1/*` через gateway |
| TLS 1.2/1.3 | **Частично** — в dev обычно HTTP; **пример** HTTPS: `docker/nginx/nginx-tls.conf.example` |
| Рабочий прототип / демо | **Да** — `docker compose`, workspace, опции ТЗ |
| Ограничений по стеку нет | **Соблюдено** |

## Сводная оценка «на сколько выполнено»

| Блок | Оценка |
|------|--------|
| **Обязательные 5 задач ТЗ** | **~95–100%** (минус «идеальный» TTS по рекомендации) |
| **Опциональные 3 пункта** | **~90%** (CSV вместо нативного XLSX — мелкий зазор) |
| **Расширенные опции** (видео / инфографика / презентация) | **~75–90%** (видео без полной серверной сборки MP4) |
| **Формальные требования к решению** | **~95%** (TLS — пример под прод, не включён в дефолтный dev) |
| **Рекомендации (Piper/Coqui)** | **Частично** |

**В целом для защиты кейса:** прототип **закрывает формулировки ТЗ** по функциям; для **максимального балла по рекомендациям** остаётся донастроить **TLS в боевом деплое**, при желании **TTS Piper/Coqui** и при необходимости **XLSX / финальная сборка видео**.
