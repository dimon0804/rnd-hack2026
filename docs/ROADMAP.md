# Дорожная карта и зоны

Кратко: что делаем дальше и кто за что отвечает. Подробнее по ролям: [`TEAM.md`](TEAM.md).

## Ближайшие шаги (продукт)

1. ~~**Индексация после загрузки**~~ — `POST /api/v1/rag/ingest` извлекает текст и индексирует; `document-service` выставляет статус `ready` или `failed`.
2. ~~**Чат с RAG**~~ — в рабочей области документа `/workspace/:id`: `POST /api/v1/rag/query` по `document_ids`, затем `POST /api/v1/ai/chat` с системным промптом по фрагментам.
3. **generation-service** — саммари, тесты, карточки, mindmap по согласованным контрактам.
4. **Качество** — лимиты размера, обработка ошибок Mistral, наблюдаемость (логи/трейсы по желанию).

## Tech Lead (ты)

- Контракты API между сервисами, **api-gateway**, **auth**, **rag** (архитектура индекса/поиска).
- Интеграция чата: схемы запросов/ответов, прокси, согласование с `ai-service`.
- Ревью PR Никиты, мерж в `develop` / `main`.

## Никита

| Зона | Задачи |
|------|--------|
| **document-service** | Парсинг файлов, надёжная передача текста в RAG, статусы в БД, edge cases (битые PDF, кодировки TXT). |
| **generation-service** | Реализация эндпоинтов генерации (саммари, тесты, карточки, mindmap), промпты по согласованию с Tech Lead. |
| **ai-service** | Доработки вызовов Mistral под новые сценарии (если не конфликтует с контрактами TL). |
| **frontend** | Чат, экраны результатов генерации, полировка UI/UX, адаптив — **основной владелец фронта** после базовой навигации/загрузки. |

Практически: после появления стабильных REST-контрактов для чата и генерации — ветки `feature/chat-ui`, `feature/generation-results` от `develop`, короткий sync при изменении OpenAPI.

## Согласование

- Новые поля в JSON — сначала в `develop`, затем фронт.
- Изменения JWT / auth — только через Tech Lead (gateway + `auth-service`).
