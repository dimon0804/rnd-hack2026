# Карта сайта и описание контента для frontend-разработчика

Документ описывает маршруты, функционал и отображаемый контент каждой страницы. Предназначен для передачи frontend-разработчику при создании нового дизайна.

---

## Общая структура приложения

- **Шапка (header)** — общая для всех страниц, sticky, с логотипом, навигацией и зоной авторизации.
- **Контент** — зависит от маршрута.

---

## Маршруты (URL)

| URL | Название | Доступ |
|-----|----------|--------|
| `/` | Главная | Все |
| `/login` | Вход | Гости |
| `/register` | Регистрация | Гости |
| `/upload` | Загрузка документов | Все (ограниченный функционал для гостей) |
| `/workspace/:documentId` | Рабочая область документа | Только авторизованные |
| `/chat` | Чат по всем документам | Редирект на `/upload` (страница временно отключена) |

---

## 1. Шапка (AppHeader)

**Поведение:**
- Sticky вверху экрана, полупрозрачный фон с blur.
- Логотип/бренд слева — клик ведёт на `/`.
- Справа — навигация и блок авторизации.

**Состояния:**
- **Гость:** ссылки «Войти», «Регистрация».
- **Авторизован:** ссылка «Загрузка», email пользователя (обрезается при длине), кнопка «Выйти».
- **До гидрации:** «…» вместо блоков.

---

## 2. Главная (`/`)

### Назначение
Лендинг: объяснение продукта и призыв к действию.

### Контент

**Hero-секция:**
- Kick-подпись: «Работа с текстами и подготовка».
- Заголовок H1: «Задавайте вопросы по своим материалам — не по интернету».
- Текст: загрузка конспектов, статей, рабочих документов; ответы опираются на ваши файлы.
- CTA-кнопки:
  - **Авторизован:** «Загрузить документы» → `/upload`.
  - **Гость:** «Войти» → `/login`, «Создать аккаунт» → `/register`.
- Подсказка для гостей: «После регистрации вы сможете загружать файлы…».

**Карточки (3 колонки):**
1. **Ваши источники** — хранение PDF, Word, текстовых файлов в одном кабинете, отслеживание обработки.
2. **Ответы по смыслу** — система опирается на загруженные материалы, а не на случайные страницы из сети.
3. **Скоро** — краткие конспекты, карточки для повторения, проверка знаний.

---

## 3. Вход (`/login`)

### Назначение
Авторизация по email и паролю.

### Контент
- Заголовок: «Вход».
- Подзаголовок: «Войдите, чтобы видеть «Мои документы» и синхронизировать работу между устройствами».
- Форма:
  - Поля: Email, Пароль.
  - Переключатель вкладок: «Вход» / «Регистрация».
  - Кнопка «Войти».
  - Блок ошибок (красный текст).
- Ссылка «← На главную».

**Логика:** после успешного входа — редирект на `state.from` (если был) или `/upload`.

---

## 4. Регистрация (`/register`)

### Назначение
Создание аккаунта.

### Контент
- Заголовок: «Регистрация».
- Подзаголовок: «Создайте аккаунт — пароль не менее 8 символов».
- Форма:
  - Email, Пароль (мин. 8 символов).
  - Подсказка «Минимум 8 символов».
  - Кнопка «Создать аккаунт».
- Общая вкладка Вход/Регистрация, ссылка на главную.

---

## 5. Загрузка документов (`/upload`)

### Назначение
Загрузка файлов, просмотр списка «Мои документы», переход в рабочую область.

### Структура страницы

**Верхняя часть:**
- Kick: «Рабочий стол».
- Заголовок: «Загрузка документов».
- Подзаголовок: «PDF, DOCX, PPTX или TXT до 50 MB. Файл сохраняется и ставится в очередь на чанкинг и индексацию в RAG».
- Степпер: `1 Файл → 2 Индексация → 3 Рабочая область` (шаг 3 приглушён, если пользователь не авторизован).
- Текст (для авторизованных): «После загрузки откроется страница документа с кратким содержанием, чатом и тестами».

**Блок для гостей (callout):**
- Заголовок: «Загрузка без входа».
- Текст: файл можно отправить без аккаунта; чтобы видеть «Мои документы» — войти или зарегистрироваться.
- Кнопки: «Войти», «Регистрация».

**Основная область — два столбца:**

#### Левый столбец (загрузка)
- **Drop-зона:**
  - Иконка вверх, текст «Перетащите файл сюда» или «Загрузка…».
  - Подсказка: «или нажмите в эту область, чтобы выбрать с диска».
  - Форматы: «PDF · DOCX · PPTX · TXT».
  - Поддержка drag-and-drop и клика по области.
- **Ошибка загрузки** — красный текст.
- **Карточка результата** (после успешной загрузки):
  - Бейдж статуса (ГОТОВО, QUEUED, INDEXED и т.д.).
  - Имя файла, размер, MIME.
  - Сообщение от сервера, id документа.
  - Кнопка «Открыть рабочую область» (если статус ready и пользователь авторизован).

#### Правый столбец («Мои документы»)
- Заголовок «Мои документы», кнопка «Обновить».
- **Состояния:**
  - Не авторизован: «После входа здесь появятся ваши файлы…».
  - Загрузка: «Загрузка списка…».
  - Пусто: «Пока пусто — загрузите первый файл слева».
  - Есть файлы: список карточек документов.
- **Карточка документа:**
  - Название файла (до 3 строк, обрезка).
  - Мета: размер, дата загрузки.
  - Бейдж статуса (ГОТОВО, ОШИБКА и т.д.).
  - Клик по карточке → `/workspace/:id`.
- **Пагинация** (если > 8 файлов): «Назад», «1–8 из N», «Вперёд».
- **Футер:** «На главную».

## 6. Рабочая область документа (`/workspace/:documentId`)

### Назначение
Работа с одним документом: обзор, генерации, чат, тесты, карточки, mindmap, подкаст.

### Доступ
Требуется авторизация. Иначе — редирект на `/login` с `state.from` для возврата.

### Структура (три колонки)

#### Левая колонка (инфо о документе)
- Kick: «Документ».
- Название файла.
- Бейдж статуса (ГОТОВО, ОШИБКА и т.д.), опционально сообщение об ошибке.
- Блок «Ключевые темы»: список тем из AI или «Готовим обзор…» / «Темы появятся после анализа».
- Кнопка «Открыть mindmap» / «Строим карту…».

**Баннер при статусе failed:**
- «Не удалось обработать документ» + сообщение + предложение загрузить снова.

#### Центральная колонка (вкладки)

**Вкладки:**
1. Обзор  
2. Просто  
3. Кратко  
4. Отчёт  
5. Таблица  
6. Чат  
7. Тесты  
8. Карточки  
9. Презентация  
10. Mindmap  

---

##### Вкладка «Обзор»
- Заголовок: «Краткое содержание».
- Контент: 2–4 предложения сути документа (генерируется автоматически при открытии).
- Состояния: «Анализируем документ…» или готовый текст.

---

##### Вкладка «Просто»
- Описание: объяснение «как для человека», без лишних терминов.
- Заголовок «Простыми словами», кнопка «Сгенерировать».
- Текст-заполнитель до генерации; после — сгенерированный текст.

---

##### Вкладка «Кратко»
- Описание: короткий пересказ в 3–5 предложениях.
- Заголовок «Краткий пересказ», кнопка «Сгенерировать».
- Заполнитель / результат.

---

##### Вкладка «Отчёт»
- Заголовок «Формализованный отчёт», кнопка «Сформировать по шаблону».
- Пояснение: деловая справка (цель, содержание, положения, выводы, рекомендации).
- Блок с текстом отчёта (или placeholder).

---

##### Вкладка «Таблица»
- Описание: таблица в формате CSV по тексту из RAG.
- Поле «Уточнение для таблицы (необязательно)» с placeholder.
- Заголовок «Структурированные данные», кнопка «Сформировать CSV».
- Состояния: loading, ошибка, пустой CSV (предупреждение), превью CSV.
- Кнопка «Скачать .csv», подпись «Модель: …».

---

##### Вкладка «Чат»
- Подсказка: вопросы только по файлу; кнопка 🎤 — речь в текст (STT).
- Лента сообщений: «Вы» / «Ответ», пузырьки.
- Раскрывающийся блок «Источники (N фрагментов)» с отрывками и % релевантности.
- Композер: кнопка 🎤 (скрытый file input для аудио), textarea, кнопка «Отправить».

---

##### Вкладка «Тесты»
- Заголовок «Тесты», кнопка «Создать тест».
- Текст с вопросами и вариантами (A, B, В, Г) или placeholder.

---

##### Вкладка «Карточки»
- Заголовок «Карточки», кнопка «Сгенерировать».
- Сетка карточек: вопрос на лицевой стороне, ответ — при клике (flip).
- Подпись «Нажмите, чтобы перевернуть».

---

##### Вкладка «Презентация»
- Заголовок «Презентация», кнопка «Создать и скачать».
- Модель возвращает JSON со слайдами; клиент собирает один **HTML-файл** в духе Gamma (тёмный градиент, крупная типографика) и **автоматически скачивает** его; предпросмотр во фрейме; кнопка «Скачать снова».
- Открытие файла в браузере: листание стрелками / пробелом; печать в PDF при необходимости.

---

##### Вкладка «Mindmap»
- Заголовок «Интеллект-карта», кнопка «Перестроить».
- Описание: интерактивный граф связей, клик по узлу.
- Контент: граф (MindmapView) или сырой текст / сообщение об ошибке.
- Раскрывающийся блок «Исходный список» с сырым текстом от модели.

---

#### Правая колонка (быстрые действия)
- Заголовок «Быстрые действия».
- Кнопка «Создать тест».
- Блок «Аудиопересказ (сценарий)»:
  - Выбор тона (Популярный / Научный).
  - Выбор темпа (Медленнее / Норма / Быстрее).
  - Кнопка «Сгенерировать диалог».
  - Превью сценария (реплики «Алексей:», «Мария:»).
  - Кнопки «Озвучить в браузере», «Стоп».
  - Подсказка про Web Speech API и голоса.
- Кнопка «Презентация (слайды)» (переход на вкладку и генерация).
- Кнопка «Mindmap (граф)».
- Ссылка «На главную».

---

## 7. Чат по всем документам (`/chat`)

**Текущее состояние:** редирект на `/upload`.

**Запланированный контент (есть компонент ChatPanel):**
- Заголовок «Вопросы по загруженным файлам».
- Подзаголовок: ответ строится на ваших документах со статусом «готово».
- Блок «N файлов в индексе», ссылка «Загрузить ещё».
- Баннер при отсутствии готовых документов.
- Лента сообщений пользователь / ассистент.
- Блок «Источники» с фрагментами.
- Композер: кнопка 🎤 (STT), textarea, «Отправить».
- Запросы идут по всем проиндексированным документам пользователя.

---

## Переиспользуемые элементы и состояния

### ProcessingOverlay
- Полупрозрачный оверлей с индикатором загрузки при `busy` (загрузка файла и т.д.).

### Бейджи статусов документов
- ГОТОВО, QUEUED, INDEXED, ОШИБКА и др. (маппинг через `documentStatusRu`).

### Ошибки и подсказки
- Блоки ошибок — красный цвет.
- Предупреждения — жёлтый/оранжевый.
- Muted-текст — приглушённый цвет.

---

## API и зависимости

- Запросы идут через gateway (`/api/v1/...`).
- Авторизация: Bearer-токен в заголовке.
- Основные эндпоинты: auth, documents (upload, list, get), rag (ingest, query, chunks), ai (chat, transcribe, extract-table).

---

## Рекомендации для нового дизайна

1. Сохранить все маршруты и функциональные блоки.
2. Уточнить приоритеты: главная vs рабочая область vs загрузка.
3. Определить адаптив: поведение трёхколоночной раскладки на мобильных.
4. Учесть тёмную тему (текущие CSS-переменные: `--bg`, `--text`, `--muted`, `--accent`, `--border`, `--danger`).
5. Обеспечить доступность: aria-labels, фокус, контраст.
